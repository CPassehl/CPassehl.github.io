This is an attempt to talk about the curious
and sometimes vexing cases of 
NaN, null and undefined.
In a nutshell, NaN (Not-a-Number)'is a special value used to denote an unrepresentable value'.
In certain instances, NaN will return a false value with regard to Boolean expressions.
Enter "NaN = NaN" into your console and NaN will be returned.
Enter "NaN == NaN" or "NaN === NaN"(== is the equality operator,
=== is the identity operator) and either will return false. Crazy, right?

Null and undefined also have unique idiosyncrasies. Null and undefined are each expressed
as a javascript primitive, meaning that either'is data that is not an object and has no methods'.

When first learning javascript, it is sometimes confusing (!) to understand
that NaN does not equal undefined or null, and that none of these values equal each other.
